# Voyd 0.3.1

Voyd `0.3.1` focuses on making the `0.3` full-stack release easier to use in
real applications. It improves enum construction and matching, hardens SSR and
hydration, fixes locally linked project scaffolding, and closes compiler and
editor correctness gaps found after `0.3.0`.

## Highlights

- Construct nominal objects, values, enum and union members, aliases, and
  qualified names with fieldwise parenthesized syntax. Match patterns can also
  resolve unqualified enum and union members from context.
- Render the same event-producing VX view on the server and in the browser.
  Server rendering now owns retained callbacks and releases them on success,
  failure, or cancellation.
- Produce standards-correct SSR output for void and raw-text elements,
  textareas, carriage returns, leading newlines, and supported form properties,
  with stricter browser/server normalization and hydration metadata.
- Bootstrap projects from a locally linked CLI without mixing current templates
  with older published runtime packages. Local scaffolds use a complete
  `file:`-linked Voyd dependency graph; pass `--published` to use registry
  packages explicitly.
- Discover imports introduced by syntax macros and preserve static method access
  through type aliases and re-export chains.
- Highlight trailing callback `do` clauses and the current Voyd language surface
  consistently in VS Code and other TextMate grammar consumers.

## Breaking changes

- Custom `VxDomRenderer` implementations must provide `detach(): void` in
  addition to `dispose()`. `detach()` releases listeners, render state, and
  retained handlers without clearing the container; `dispose()` also removes
  the rendered DOM.
- A program produced by `map_model(program, map)` cannot adopt an SSR hydration
  model because that mapping only describes the child-to-parent direction.
  Hydrated applications must use the labeled overload and provide the reverse
  mapping: `map_model(program: child, map: to_parent, hydrate: to_child)`.
- VX rendering now rejects trees that cannot have equivalent browser and server
  semantics. Dynamic HTML tag and attribute names must be lowercase; SVG
  descendants preserve their case-sensitive tag and attribute names. Void
  elements cannot have children, structured CSS values cannot contain `;`,
  `!`, or control characters, and server-rendered DOM properties are limited
  to the documented form-control combinations. Use `attr` for ordinary HTML
  attributes and classes for complex styling.
- Server-rendered HTML is now standards-correct for void elements, raw-text
  elements, textareas, carriage returns, and leading newlines. Exact-output
  snapshots may need updating. Hydration scripts are inserted before
  `</body>`, include root metadata, and hydration-model serialization errors now
  stop the response instead of silently emitting `null`.

## Install

```sh
npm i -g @voyd-lang/cli@0.3.1
```

Update all directly consumed Voyd packages together.
