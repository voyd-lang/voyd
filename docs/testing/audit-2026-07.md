# Test Organization Audit — July 2026

## Baseline

Before this migration the repository had roughly 314 test files, 2,100 test
cases and 69,600 lines of tests. `packages/compiler` accounted for 203 files
and roughly 1,219 cases. Recent PRs spent seven to eight and a half minutes in
the affected-package test step; dependency installation was about 12–16
seconds.

The former `apps/smoke` package contained 29 TypeScript files spanning language
semantics, SDK integration, host adapters, packaging, VX DOM, web and
benchmarks. Its name and flat placement obscured canonical ownership.

## Changes Made

The maintained [repository-wide inventory](test-inventory.json) records an
owner, disposition and rationale for every current test file. The historical
[migration disposition ledger](audit-dispositions-2026-07.md) separately
records the decision and surviving coverage for every former `apps/smoke`
test, including files that were merged or removed.

The inventory is a reviewed source of truth rather than a directory-derived
report. All 203 compiler test files were inspected for the contract they
protect: 71 are explicitly implementation-only; 52 name directly corresponding
portable cases with no additional gap identified; 24 name representative cases
and an explicit remaining gap; and 56 record portable backlog without claiming
coverage. The audit extracted intersection and recursive types, static access,
overload ranking, closures, collection/subscript behavior, optimized
default/alias/evaluation semantics, modules/macros, diagnostic spans, HTML,
continuations and effect diagnostics into conformance.

- Replaced `apps/smoke` with dedicated conformance, integration and performance
  workspaces under `tests/`.
- Added a compiler-neutral adapter and manifest with 118 portable expectations
  across syntax, typing, values, effects, intersection/static-access semantics,
  module visibility, traps, host interactions, and basic Wasm ABI.
- Preserved all compiler implementation tests in `packages/compiler`.
- Migrated cross-package host, packaging, web and VX tests to
  `tests/integration`.
- Moved vtrace benchmarks, the scalar-aggregate benchmark fixture and the
  external vtrace regression to opt-in `tests/performance`.
- Removed the std-owned LocalRng duplicate and two orphan fixtures.
- Removed redundant CLI companion-module, nested compile and default
  `node_modules` subprocess scenarios; canonical compiler/SDK coverage remains.
- Removed duplicate SDK runtime-diagnostic and optimizer-semantic checks.
- Removed duplicate effect metadata helper assertions from integration; SDK
  remains their canonical owner.
- Split PR CI into unit, conformance and integration lanes, set explicit Turbo
  concurrency, cancelled superseded runs and narrowed dist CLI validation for
  non-CLI changes.
- Added test-ownership, regression-evidence and runtime prompts to the pull
  request template.

The current inventory is reproducible with:

```sh
npm run test:audit:update
```

`npm run test:audit` rejects unclassified or stale test files so the inventory
remains complete as the repository changes.

`npm run test:audit:update` preserves reviewed decisions and adds new files as
`needs-review`; it does not bless placement based on the current directory.

## Validation And Timing

Post-migration local validation on the same workstation produced:

- portable conformance: 118 manifest cases plus two harness checks in about 18
  seconds with the two-worker CI cap;
- cross-package integration: 128 cases in about 78 seconds;
- packaged CLI smoke: 3 cases in about 5 seconds;
- full source-mode CLI e2e after duplicate removal: 14 cases in about 38
  seconds; and
- complete uncached default workspace sweep: about 92 seconds.

The pre-migration uncached workspace sweep was about 74 seconds on this
workstation. The new isolated workspaces reduce measured aggregate CPU work
(about 578 to 520 user-seconds in sampled runs) but increase local wall time
because the portable and integration boundaries are now scheduled as separate
packages. PR CI runs these lanes on separate workers; its former seven-to-eight
minute serialized affected step is the primary optimization target. The PR's
actual Actions timings should be treated as the authoritative before/after
wall-clock comparison.

## Deliberate Retentions

- Compiler units remain co-located and should continue growing when they are
  the cheapest precise protection for an implementation invariant.
- Std and other Voyd package tests remain beside their source.
- Integration effect and task fixtures remain batched because one expensive
  compilation serves many distinct host/runtime boundaries.
- CLI `--pkg-dir` compile and test cases remain separate because the two
  commands resolve their roots through different wiring.

## Remaining Opportunities

These were audited but not forced into this migration:

- Split the mixed `wasm-validation` integration file into compiler mechanism,
  portable Wasm ABI and VX integration owners.
- Move SDK web-server/default-adapter compositions into integration when doing
  so can reuse existing compiled fixtures rather than duplicate them.
- Memoize repeated VX fixture compilations and consolidate web integration
  sources by subsystem.
- Introduce duration-aware codegen sharding once CI publishes reliable
  per-file history.

These items are lower confidence or require new infrastructure; retaining them
is safer than deleting unique coverage merely to make the migration exhaustive.
